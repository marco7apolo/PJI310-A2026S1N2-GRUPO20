Visão Geral

O SGO (Sistema de Gestão de Oficina) é uma aplicação web desenvolvida em Django para gerenciar oficinas de reparo de eletroeletrônicos. O sistema atende à demanda de pequenos empreendimentos do setor, muitos atuando na informalidade, oferecendo ferramentas para cadastro de clientes, equipamentos, técnicos e ordens de serviço (reparos), além de dashboards e geração de laudos técnicos em PDF.

Contexto Social
  • Brasil é o 5º maior produtor de lixo eletrônico do mundo
  • Apenas ~3% dos resíduos eletrônicos são reciclados
  • 37,5% da população ocupada no Brasil está na informalidade
  • Metodologias centradas no usuário (Design Thinking)

Jornada de Desenvolvimento

Fase 1: Configuração Inicial e MySQL

Configuração inicial do projeto Django, desenvolvimento local com MySQL, implementação dos modelos e templates.

Fase 2: Migração para PostgreSQL/Supabase

Desafios:

  • Erro de conexão IPv6: Cannot assign requested address
  • Erro de Pooler: Tenant or user not found

Resultado: Migração abandonada, migrou para Neon.

Fase 3: Migração para Neon (Sucesso)

Configuração do Neon com Pooled connection para melhor performance serverless.

# String de conexão Neon postgresql://neondb_owner:PASSWORD_REMOVIDO@ep-royal-pond-ap7l2s2n-pooler.c-7.us-east-1.aws.neon.tech/neondb?sslmode=require
Fase 4: Configuração Django para Produção

Implementação de dj-database-url, variáveis de ambiente, DEBUG=False.

Fase 5: Deploy na Vercel

Configuração de variáveis de ambiente, correção de erros de autenticação, deploy bem-sucedido.
